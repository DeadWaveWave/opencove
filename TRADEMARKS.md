# OpenCove Trademarks & Brand Guidelines

OpenCove is open source and distributed under the MIT License. The MIT License grants copyright permissions for the code, but it **does not** grant trademark rights.

This document explains how you may use the **OpenCove** name, logos, and other brand features ("**Marks**").

- Contact: **deadwavewave@gmail.com**

## Allowed (no permission needed)

You may, without prior permission:

- Use the word **OpenCove** in text to truthfully refer to the Project (nominative use).
- Say that your work is **compatible with OpenCove** (for example, "plugin for OpenCove"), as long as it does not imply official endorsement.
- Link to the Project and use **unmodified** screenshots to discuss, review, or document OpenCove.

## Not allowed (permission required)

You must get written permission before you:

- Use any Mark as the **name, logo, or primary branding** of your own product, company, or service.
- Distribute a **modified** version of OpenCove using the **OpenCove** name, app icon, or logo in a way that could confuse users about whether it is an official build.
- Use the Marks in a way that suggests the Project steward (or OpenCove) **sponsors, endorses, or certifies** you or your product.
- Register or use domains/social handles that are confusingly similar to OpenCove (for example, `opencove-*.com`), in a way that could mislead users.

## Forks and redistributions

If you redistribute a fork or modified build, you should:

- Use a **distinct name** and **distinct branding** (including app name and icon).
- Clearly state that your build is **not** the official OpenCove distribution.
- Avoid using OpenCove logos/brand assets in your distribution metadata (website, store listing, screenshots, etc.) in a way that implies endorsement.

## Plugins and extensions

You may reference OpenCove to describe compatibility (e.g., "X for OpenCove"). Avoid using **OpenCove** as the primary brand of your extension, and do not claim your extension is official unless it is.

## Notes

This document is a project policy intended to reduce user confusion. It is not legal advice.
